# DataViz documentation

The GitHub Pages site is generated by Sphinx from `docs/source` and styled
with `sphinx_rtd_theme`. The hand-built `website/` directory is a legacy
prototype and is not part of the publication workflow.

## Local build

```bash
python -m pip install -e ".[docs]"
python -m sphinx -W --keep-going -b html docs/source docs/build/html
```

The resulting site is available at `docs/build/html/index.html`.

## Link validation

```bash
python -m sphinx -W --keep-going -b linkcheck docs/source docs/build/linkcheck
```

## API regeneration

```bash
sphinx-apidoc --force --separate --module-first --output-dir docs/source/api dataviz
```

## GitHub Pages

The `.github/workflows/docs.yml` workflow validates pull requests and deploys
pushes to `main` through GitHub's official Pages artifact and deployment
actions. Configure the repository's Pages source as **GitHub Actions**.
